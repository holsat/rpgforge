# Mythos

An AI-native worldbuilding writer's app — manuscript editor, codex, timeline, atlas, story weaver, and a Muse AI assistant. Built with **React + Vite + Tauri 2** for native desktop on macOS, Windows, and Linux. Mobile (iOS/iPadOS, Android) is supported via Tauri's mobile targets.

> ⚠️ This is a **starting-point scaffold**, not a finished application. The UI is fully built and on-brand; persistence, real AI integration, file I/O, and code-signing still need to be implemented. See "Roadmap" below.

---

## Prerequisites

You need three toolchains installed before this project will build:

1. **Node.js ≥ 18** — https://nodejs.org/
2. **Rust** (stable) — https://rustup.rs/
3. **Platform build tools:**
   - **macOS:** Xcode Command Line Tools — `xcode-select --install`
   - **Windows:** Microsoft C++ Build Tools + WebView2 (usually pre-installed on Win11)
   - **Linux:** see the [Tauri Linux prerequisites](https://tauri.app/start/prerequisites/#linux) for your distro (typically `webkit2gtk`, `librsvg`, `libsoup`, `build-essential`).

Verify Rust + Tauri are healthy:
```bash
rustc --version
cargo --version
```

## First-time setup

```bash
cd mythos-app
npm install
```

This installs the React/Vite frontend deps and the `@tauri-apps/cli` binary. Cargo will fetch Rust crates the first time you run `tauri dev` (this takes 5–10 minutes the first time).

---

## Run in development

```bash
npm run tauri:dev
```

This starts Vite on `http://localhost:1420` and launches a native window pointing at it. Hot-reload works for the frontend; Rust changes recompile automatically.

To preview the frontend in a browser only (no native window, no Rust):
```bash
npm run dev
```

---

## Build for production

Generate a real installer for your current OS:

```bash
npm run tauri:build
```

Outputs land in `src-tauri/target/release/bundle/`:
- **macOS:** `.app` and `.dmg`
- **Windows:** `.msi` and `.exe`
- **Linux:** `.deb`, `.rpm`, `.AppImage`

> You'll need to generate icons first (see below), or `tauri build` will fail.

---

## Generate app icons

Tauri requires a full icon set. Place a 1024×1024 PNG of your logo somewhere, then:

```bash
npm run tauri icon path/to/mythos-logo.png
```

This writes `src-tauri/icons/32x32.png`, `128x128.png`, `128x128@2x.png`, `icon.icns` (macOS), and `icon.ico` (Windows).

## Mobile (iOS / iPadOS / Android) — optional

Tauri 2 supports mobile, but it's a separate flow with its own toolchain.

### iOS / iPadOS
Requires macOS, Xcode, and an Apple Developer account.
```bash
npm run tauri ios init
npm run tauri ios dev
```
You'll need to configure code signing in Xcode and submit through TestFlight / App Store.

### Android
Requires Android Studio + NDK.
```bash
npm run tauri android init
npm run tauri android dev
```

## Calling Rust from React

`src-tauri/src/lib.rs` already exposes two demo commands. Call them from any component:

```jsx
import { invoke } from '@tauri-apps/api/core';

// In a useEffect or button handler:
const reply = await invoke('greet', { name: 'Elara' });
console.log(reply); // "Welcome, Elara. The Ledger remembers."
```

Add new commands by writing `#[tauri::command] fn my_thing(...)` and listing it in the `invoke_handler![...]` macro.

### Polish

- [ ] Auto-updater via `tauri-plugin-updater`
- [ ] Error handling, loading states, empty states

## Design system

The visual system is defined in `src/styles/mythos.css`. Key tokens:

- **Palette:** charcoal void (`--bg-0: #0a0a0a`) with antique gold accents (`--mythos-gold: #D4AF37`)
- **Type:** Cinzel (display) / EB Garamond (serif) / Inter (sans) / JetBrains Mono (mono)
- **Density:** generous spacing, deliberate ornamentation, no emoji

If you swap tokens, every screen updates.
